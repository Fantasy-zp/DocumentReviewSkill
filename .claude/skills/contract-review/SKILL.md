# 合同智能审核 Skill

## 触发条件

当用户执行以下操作时触发此 Skill：
- 使用 `/contract-review` 命令
- 上传合同文件并要求审核
- 询问合同相关的风险问题

## 审核流程 SOP

### 第一步：合同基本信息识别

1. **识别合同类型**
   - 劳动合同
   - 销售/采购合同
   - 服务合同
   - 租赁合同
   - 借款合同
   - 合作/框架协议
   - 其他

2. **提取关键信息**
   - 合同双方（甲方、乙方）
   - 签订日期
   - 合同期限（起止时间）
   - 合同金额/对价
   - 合同标的

### 第二步：条款风险扫描

按以下维度逐一检查，参考 `reference/risk-checklist.md`：

#### 高风险条款（必须标注）
- [ ] 单方免责条款 - 一方完全免除责任
- [ ] 无限连带责任 - 承担过重的连带义务
- [ ] 自动续约陷阱 - 未通知则自动续约且难以解除
- [ ] 单方解除权失衡 - 对方可随时解除，己方不可
- [ ] 违约金过高 - 超过损失的30%
- [ ] 知识产权全归属 - 不合理的IP归属约定
- [ ] 竞业限制过宽 - 范围、时间、地域不合理

#### 中风险条款（建议关注）
- [ ] 付款条件苛刻
- [ ] 验收标准模糊
- [ ] 保密期限过长
- [ ] 管辖地对己方不利
- [ ] 不可抗力定义过窄
- [ ] 通知条款不明确

#### 低风险/建议优化
- [ ] 条款表述模糊
- [ ] 缺少争议解决条款
- [ ] 缺少合同变更条款

### 第三步：合规性检查

1. **必备条款检查**
   - 是否有明确的合同主体
   - 是否有标的物/服务描述
   - 是否有价款及支付方式
   - 是否有履行期限
   - 是否有违约责任
   - 是否有争议解决方式

2. **法律合规检查**
   - 是否存在违反强制性法规的条款
   - 格式条款是否履行提示义务
   - 是否涉及需要审批/备案的事项

### 第四步：生成审核报告

使用 `assets/report-template.md` 模板，输出结构化报告：

```markdown
## 合同审核报告

### 基本信息
- 合同类型：
- 合同双方：
- 合同金额：
- 合同期限：

### 风险摘要
- 高风险：X 项
- 中风险：X 项
- 建议优化：X 项

### 风险详情
[逐条列出风险点、所在位置、风险说明、修改建议]

### 审核结论
[通过/建议修改后签署/不建议签署]
```

## 输出规范

1. **风险等级标识**
   - 🔴 高风险 - 强烈建议修改
   - 🟡 中风险 - 建议关注或协商
   - 🟢 低风险/优化建议

2. **修改建议要求**
   - 每个风险点必须给出具体修改建议
   - 建议应当可直接采纳使用
   - 必要时提供替代条款文本

3. **语言要求**
   - 使用专业但易懂的中文
   - 避免过度法律术语
   - 给出风险的实际影响说明

## 使用外部能力

当需要深度分析时，可调用 `scripts/llm_client.py` 使用 LLM 能力：
- 复杂条款的语义理解
- 上下文关联风险分析
- 生成修改建议文本

## 注意事项

1. 本 Skill 提供的是辅助审核意见，不构成法律建议
2. 重大合同建议配合专业律师审核
3. 审核结果应结合具体业务场景判断
